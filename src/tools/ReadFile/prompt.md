# read_file

## DESCRIPTION

读取文件中的正文文本，支持".docx"文件和".py", ".ts", ".tsx", ".js", ".jsx", ".json", ".md", ".txt"文件的读取。

## PROMPT

### WHEN_TO_USE
当用户需要读取、检查、总结或分析 `.docx`、`.py`、`.ts`、`.tsx`、`.js`、`.jsx`、`.json`、`.md`、`.txt` 文件正文内容时，使用这个工具。

适合用于查看代码文件、Markdown 文档、纯文本文件、JSON 配置文件，以及只需要提取普通正文段落的 Word 文档。

### WHEN_NOT_TO_USE
不要把它用于 `.doc`、PDF、图片、扫描件、音视频文件、压缩包，或其他不在支持列表中的文件格式。
不要把它用于需要严格还原 Word 版式的任务，例如字体、字号、颜色、分页、图片、页眉、页脚、批注、修订记录或表格结构分析。读取 `.docx` 时，本工具只提取普通正文段落。

### INPUT_RULES
- `file_path`：必填，要读取的文件绝对路径。必须指向真实存在的文件。
- `offset`：可选，从第几行开始读取，默认为 `1`。应使用大于等于 `1` 的整数。
- `limit`：可选，最多读取多少行。不填时使用工具默认读取行数；文件较长时可通过多次调用继续读取。该值最大为200

### OUTPUT_RULES
- 返回内容中的每一行通常带有行号前缀，例如 `12: content`。
- 行号只用于定位，不属于文件原始内容。
- 解释代码或文档时可以引用行号。
- 复写、复制或生成文件内容时，不要把行号前缀写入正文。
- `truncated=true` 表示本次结果未覆盖完整文件，可能是因为还有后续行，或内容超过字符上限被截断。
- 在需要完整理解文件内容时，如果 `truncated=true`，应继续使用新的 `offset` 读取后续内容后再做最终结论。

### LIMITS
- 支持的文本文件格式包括 `.py`、`.ts`、`.tsx`、`.js`、`.jsx`、`.json`、`.md`、`.txt`。
- 支持 `.docx`，但只读取普通正文段落。
- 不保留 `.docx` 的复杂版式、图片、表格、批注、页眉页脚或修订信息。
- 单次读取存在最大行数和最大字符数限制；长文件需要分段读取。
- 如果文件读取失败，应根据错误信息判断原因，不要在路径或格式没有变化的情况下反复调用。
