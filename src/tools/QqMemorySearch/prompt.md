# qq_memory_search

## DESCRIPTION
从指定 QQ 群的最新消息开始，向更早的消息分页读取历史记录。

## PROMPT

使用本工具读取指定 QQ 群的原始聊天记录。

只有在当前问题缺少必要的群聊上下文时才调用本工具。调用前必须先确定检索目标，例如：
- 查找被回复消息的 message_id；
- 查找当前消息附近的上下文；
- 查找明确指定时间范围内的消息；
- 查找某条相关图片消息的 image_urls。

参数规则：
1. group_index 必须使用当前交互提供的 QQ 群号，不得猜测其他群号。
2. 第一次调用时 cursor 留空。
3. 为减少调用次数，limit 使用 100。
4. 继续翻页时，cursor 必须原样使用上一次返回的 next_cursor，不得自行计算或重复使用旧 cursor。
5. 消息整体从最新记录向更早记录翻页；单页内第一条消息最旧，最后一条消息最新。

每次获得结果后，按照以下顺序判断：

1. 已找到目标 message_id、目标图片 URL 或足以回答问题的上下文：
   立即停止调用并输出相关证据。

2. 用户只需要“最近的聊天内容”或当前消息附近的普通上下文：
   默认只读取一页，然后停止；不得因为 has_more=true 自动继续。

3. 用户指定了时间范围：
   当本页最早一条消息的 datetime 已经早于时间范围下界时，说明时间窗口已经覆盖，立即停止。

4. 返回 has_more=false 或 next_cursor 为空：
   立即停止。

5. 已经读取 1000 条消息：
   无论是否找到目标，都必须停止，并输出“在最近 1000 条消息中未找到明确相关信息”。

只有同时满足以下全部条件时才允许继续调用：
- 当前检索目标尚未找到；
- 当前证据确实不足；
- has_more=true；
- next_cursor 不为空；
- 尚未读取满 600 条消息。

禁止仅仅因为 has_more=true 就继续调用。
禁止重复检索已经读取过的 cursor。
禁止为了理解普通文字消息而查找图片 URL。

本工具只能返回图片 URL，不能理解图片内容。是否使用其他图片分析工具由上层节点决定，本工具本身不要要求调用 imageread。

完成检索后只输出与当前问题直接相关的消息、发送者、时间、message_id、reply 和必要的 image_urls，不要复述无关记录。
