# memory_search

## DESCRIPTION
逐层读取 `memory` 目录内的 YAML 长期记忆文件。
## PROMPT
当需要查找已经整理好的长期记忆时，使用本工具。
本工具只读取 `memory` 目录中的 `.yaml` 或 `.yml` 文件，不读取 QQ 原始聊天记录，也不修改文件。
支持三种模式：
- `metadata`：读取文件的说明。首次接触一个记忆文件时，优先使用此模式。
- `list`：列出指定层级的子字段，不读取详细内容。不了解文件内部结构时，逐层使用此模式。
- `read`：读取指定字段的实际内容。确认字段路径后再使用此模式。

`field_path` 使用字符串列表表示字段层级。例如：
- `[]`：文件顶层
- `["members"]`：成员列表
- `["members", "1010475105"]`：指定成员下的字段
- `["members", "1010475105", "events"]`：指定成员的事件记录

建议调用顺序：
1. 使用 `metadata` 了解文件用途。
2. 使用 `list` 逐层查看可用字段。
3. 使用 `read` 获取目标字段的实际内容。
4. 如果返回 `has_more=true`，继续调用，并传入返回的 `next_cursor`。

不要猜测不存在的字段。字段不存在时，应根据错误信息返回上一级并重新使用 `list` 模式确认结构。
